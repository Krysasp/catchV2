# ViroSort

**Oligo Probe Design and Multi-Cluster Mapping Pipeline**

ViroSort is a bioinformatics pipeline designed to map oligo probes to degapped consensus sequences across multiple viral clusters, with support for adapter trimming, IUPAC degenerate base handling, and taxonomy annotation.

## Overview

ViroSort processes oligo probes by:
1. **Adapter Trimming** - Removes adapter sequences from probe 5' and/or 3' ends
2. **Multi-Cluster Mapping** - Maps probes to ALL consensus sequences with IUPAC/N tolerance
3. **Consolidation** - Consolidates matches to multiple clusters with "least degenerate" IUPAC sequence
4. **Taxonomy Integration** - Annotates probes with taxonomic information from metadata
5. **Visualization** - Generates interactive HTML reports and summary statistics

## Input Files

ViroSort requires the following input files (typically generated by Catchv2):

### 1. Degapped Consensus FASTA (`--degapped`)
- FASTA file containing consensus sequences with gaps removed
- Headers should follow format: `>CXXXX|Family|Genus|Species|Molecule_type`

### 2. Cluster File (`--cluster`)
- CD-HIT cluster file (.clstr format)
- Maps sequences to their respective clusters

### 3. Analysis File (`--analysis`)
- TSV file containing genome analysis statistics
- Used for probe mapping validation

### 4. Metadata CSV (`--metadata`)
- CSV file with taxonomy information indexed by Accession ID
- Required columns: Accession, Organism_Name, Species, Genus, Family, Molecule_type, Geo_Location, Collection_Date

### Optional Inputs

#### 5. Probes/Probe Map (`--probes`)
- TSV/CSV file mapping probes to consensus sequences

#### 6. Oligo FASTA (`--oligos`)
- FASTA file containing 26-mer oligo sequences
- Headers format: `id_CONS|cluster|position|similarity`

#### 7. Config File (`--config`)
- JSON configuration file with default parameters

## Output Files

ViroSort generates outputs in the specified output directory with the following structure:

```
output_directory/
├── fasta/
│   ├── renamed_consensus.fasta      # Consensus sequences with taxonomy headers
│   ├── renamed_probes.fasta         # Mapped probe sequences
│   └── renamed_26mer_oligos.fasta   # 26-mer oligos mapped to clusters
├── html_report/
│   └── probe_mapping_report.html    # Interactive visualization
└── summary/
    ├── summary.csv                  # Pipeline summary statistics
    ├── cluster_summary.tsv          # Per-cluster statistics
    └── probe_mapping.tsv            # Detailed probe mapping data
```

### Output FASTA Header Format
```
>ClusterID|Family|Genus|Species|Similarity|Molecule_type
```

Example:
```
>C0004|Adenoviridae|Mastadenovirus|Mastadenovirus_hylobatidae|0.000|dsDNA
```

## Usage

### Basic Command Line

```bash
# Standard pipeline with adapter trimming
python main.py \
    --degapped input_degapped.fasta \
    --cluster input_cluster.clstr \
    --probes input_probes.tsv \
    --analysis input_analysis.tsv \
    --metadata metadata.csv \
    --output output_dir \
    --adapters GTGGAGGTCGCTAGATGGTC \
    --max-clusters 100
```

### Skip Alignment Mode (High-Throughput)

For processing 26-mer oligos where cluster information is embedded in probe headers:

```bash
python main_adapter_revised.py \
    --degapped part1_degapped_input.fasta \
    --cluster part1_cdhit_clusteredseq.clstr \
    --probes part1_oligos_probemapNA26NT.csv \
    --analysis part1_oligos_anaM6NA26NT.tsv \
    --metadata ncbi_virus_noncov_metadata.csv \
    --output part1_output \
    --skip-alignment \
    --oligos part1_oligosM6NA26NT.fasta \
    --max-clusters 1533
```

### Key Parameters

| Parameter | Description | Required | Default |
|-----------|-------------|----------|---------|
| `--degapped` | Path to degapped consensus FASTA | Yes | - |
| `--cluster` | Path to cluster file (.clstr) | Yes | - |
| `--analysis` | Path to analysis TSV file | Yes | - |
| `--metadata` | Path to metadata CSV file | Yes | - |
| `--output` | Output directory path | Yes | - |
| `--probes` | Path to probe map TSV file | No | None |
| `--oligos` | Path to 26-mer oligo FASTA | No | None |
| `--adapters` | Adapter sequence(s) to trim | No | 4 known adapters |
| `--skip-alignment` | Skip sequence alignment; use cluster from header | No | False |
| `--max-clusters` | Maximum clusters to process | No | All |
| `--trim-len` | Bases to trim if adapter not detected | No | 20 |
| `--max-unmatched` | Maximum unmatched bases allowed | No | 6 |

## Processing Modes

### 1. Standard Mode (main.py)
- Full pipeline with taxonomic annotation
- NCBI Entrez integration (optional)
- Excel output generation
- Interactive HTML visualization

### 2. Skip-Alignment Mode (main_adapter_revised.py)
- Extracts cluster number directly from probe header
- Faster processing for high-throughput scenarios
- No sequence alignment required
- Supports 26-mer oligo FASTA input

## Output Summary

The `summary.csv` file contains aggregate statistics:

| Column | Description |
|--------|-------------|
| Group_Type | Cluster grouping type |
| Group_Name | Molecule type (dsDNA, ssRNA(+), etc.) |
| Molecule_Type | Sequence molecule classification |
| Total_Consensus | Number of consensus sequences |
| Unique_Clusters | Number of unique clusters |
| Total_Probes | Total probes processed |
| Probes_Matched | Successfully mapped probes |
| Avg_Mismatch_Threshold | Average mismatches allowed |

## Molecule Types

ViroSort recognizes the following molecule types:
- `dsDNA` - Double-stranded DNA
- `ssRNA(+)` - Positive-sense single-stranded RNA
- `ssRNA(-)` - Negative-sense single-stranded RNA
- `ssDNA(+)` - Positive-sense single-stranded DNA
- `ssDNA(-)` - Negative-sense single-stranded DNA
- `ssDNA(+/-)` - Ambiguous single-stranded DNA

## Performance Notes

- **Parallel Processing**: Uses multiprocessing with configurable thread count
- **Memory Efficient**: Processes clusters in batches when `--max-clusters` is specified
- **Skip-Alignment Mode**: Significantly faster for 26-mer oligo mapping

## Integration with Catchv2

When integrated into the Catchv2 repository:
1. Catchv2 generates input files (degaapped FASTA, cluster files, probemaps)
2. ViroSort processes these inputs to map probes and generate outputs
3. Results can be fed back into Catchv2 for downstream analysis
